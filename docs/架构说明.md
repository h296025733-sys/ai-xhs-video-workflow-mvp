# 架构说明

## 总体思路

项目把“自然语言任务”和“视频处理脚本”分开：

- OpenClaw / 企业 IM 负责接收中文需求。
- 包装脚本负责解析链接、BGM、dry-run、起始秒数、输出目录等意图。
- Playwright / Chrome CDP 负责在已登录浏览器里搜索小红书作者、进入主页并采集作品。
- XHS-Downloader API/CLI/MCP 负责把小红书作品解析或下载到本地。
- FastAPI 负责创建任务、管理素材、调用 FFmpeg。
- 本地 outputs 目录保存结果，不进入 Git。

## 核心模块

| 模块 | 作用 |
| --- | --- |
| `app/main.py` | FastAPI 应用入口 |
| `app/quick_bgm/router.py` | quick-bgm API 和本地演示页面 |
| `app/quick_bgm/media.py` | FFmpeg / ffprobe 调用 |
| `app/quick_bgm/store.py` | 本地 JSON 状态与 BGM 库 |
| `app/quick_bgm/xhs_adapter.py` | 单条小红书链接导入适配 |
| `app/quick_bgm/xhs_creator_importer.py` | 作者主页和作品预览适配 |
| `scripts/quick_bgm_automation.py` | 通用 CLI |
| `scripts/openclaw_quick_bgm.py` | 自然语言消息包装脚本 |
| `scripts/openclaw_xhs_author.py` | 作者主页自然语言入口，支持小红书名/号搜索 |
| `scripts/xhs_author_auto_pipeline.py` | Chrome CDP 主页采集、时间/点赞筛选、批量处理 |
| `scripts/openclaw_xhs_author_direct_v2.py` | 复用筛选报告生成多 BGM 版本的实验路径 |

## 关键边界

公开版不包含真实登录态、真实素材、真实 OpenClaw 配置。作者主页自动化依赖你本机已登录 Chrome 和平台页面结构，仓库只保留脱敏后的可复用脚本与配置示例。

## 作者主页链路

```text
中文消息
  -> openclaw_xhs_author.py
  -> Playwright 连接 Chrome CDP
  -> 小红书搜索页定位作者
  -> 作者主页滚动采集作品链接
  -> preview_note_link 读取作品元数据
  -> 最近 N 天 / 点赞 N 以上 / 视频信号筛选
  -> openclaw_quick_bgm.py
  -> FastAPI quick-bgm 任务
  -> FFmpeg 输出成品
```
