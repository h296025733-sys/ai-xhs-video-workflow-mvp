# 常见问题

## 这是完整自动发布系统吗？

不是。当前公开版是本地 MVP 和工作流骨架，不包含 TikTok 自动发布。

## 作者主页自动化已经实现了吗？

已整理公开版脚本：`scripts/openclaw_xhs_author.py` 和 `scripts/xhs_author_auto_pipeline.py`。

它依赖你本机已登录的小红书 Chrome，并通过 `XHS_CDP_URL=http://127.0.0.1:9222` 连接浏览器。公开仓库不会内置私人 cookie，所以你需要自己启动带调试端口的 Chrome。

## 为什么作者主页仍然可能失败？

小红书页面结构、登录态、风控策略都会变化。脚本已经做了 DOM、网络响应、`window.__INITIAL_STATE__` 多路提取，但仍建议把它当成本地工作流，不要当成永久稳定的商业采集服务。

## 没有 XHS-Downloader 能用吗？

可以。你仍然可以上传本地视频和本地 BGM，使用 FFmpeg 完成换 BGM。

如果要处理小红书作品链接，建议配置 XHS-Downloader API/CLI/MCP，或先人工下载视频再走本地视频路线。

## BGM 搜索失败怎么办？

可以手动上传 BGM，或换更明确的搜索词，例如“韩国 TikTok OOTD 热门歌 2026”。

## 为什么不提交 outputs？

outputs 里通常是处理后视频、封面、报告和日志，可能包含真实素材、个人信息或版权内容，不适合进入公开仓库。
