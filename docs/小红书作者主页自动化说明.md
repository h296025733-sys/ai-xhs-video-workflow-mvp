# 小红书作者主页自动化说明

这条链路对应真实使用里最常见的一句话：

```text
找小红书号 YOUR_XHS_ID，最近7天内点赞100以上的视频，取5条，
BGM 用韩国 TikTok OOTD 热门歌，从第20秒开始。
```

它不是简单爬网页，而是复用一个已经登录的小红书 Chrome 浏览器，让 Playwright 通过 CDP 操作页面。这样做的好处是：登录态留在你本机浏览器里，不需要把 cookie 写到代码或 GitHub。

## 工作流

```text
中文消息
  -> openclaw_xhs_author.py 解析作者、时间、点赞、BGM 条件
  -> Playwright 连接 Chrome CDP
  -> 搜索小红书号/昵称，进入作者主页并二次校验
  -> xhs_author_auto_pipeline.py 滚动主页采集作品链接
  -> 结合 DOM、网络响应、window.__INITIAL_STATE__ 提取 note_id / xsec_token
  -> preview_note_link 读取作品元数据
  -> 按发布时间、点赞数、视频信号筛选
  -> openclaw_quick_bgm.py 批量下载/换 BGM
```

## 准备 Chrome CDP

Windows 示例：

```powershell
& "C:\Program Files\Google\Chrome\Application\chrome.exe" `
  --remote-debugging-port=9222 `
  --user-data-dir="D:\workspace\xhs-cdp-profile"
```

打开后先登录小红书，再运行脚本：

```powershell
$env:XHS_CDP_URL = "http://127.0.0.1:9222"
```

## 只解析参数

```powershell
.\.venv\Scripts\python.exe .\scripts\openclaw_xhs_author.py `
  --message "找小红书号 YOUR_XHS_ID，最近7天内点赞100以上的视频，取5条，BGM用韩国 TikTok OOTD 热门歌，从第20秒开始" `
  --parse-only
```

## 正式执行

```powershell
.\.venv\Scripts\python.exe .\scripts\openclaw_xhs_author.py `
  --api-base http://127.0.0.1:8004 `
  --message "找小红书号 YOUR_XHS_ID，最近7天内点赞100以上的视频，取5条，BGM用韩国 TikTok OOTD 热门歌，从第20秒开始，正式生成"
```

## 可识别的中文条件

| 说法 | 解析含义 |
| --- | --- |
| `最近7天内` | 发布时间在最近 7 天 |
| `最近30天` | 发布时间在最近 30 天 |
| `7-15天前` | 排除太新和太旧的作品 |
| `点赞100以上` | 只保留点赞数 >= 100 的作品 |
| `取5条` | 最多处理 5 条 |
| `BGM用韩国 TikTok OOTD 热门歌` | 传给 BGM 搜索/下载链路 |
| `从第20秒开始` | BGM 从第 20 秒截取 |
| `先看看` / `dry-run` | 只预览，不生成成品 |

## 常见卡点

- Chrome 没用 `--remote-debugging-port=9222` 启动。
- 当前 Chrome 没登录小红书。
- 搜索结果里没有匹配的小红书号/昵称。
- 小红书页面结构变化，DOM 提取规则需要调整。
- 作品需要更完整登录态，XHS-Downloader API 没拿到详情。
- 作品是图文，不是视频。
- 点赞数或发布时间字段没有返回，脚本会尽量避免误杀，但最终仍建议人工复核。

## 安全边界

这个脚本会在运行态生成筛选报告和候选链接，其中可能包含小红书临时参数。默认输出在 `outputs/` 和 `.runtime/`，已经被 `.gitignore` 排除。不要把这些运行产物上传 GitHub。
